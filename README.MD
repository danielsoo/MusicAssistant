# MusicAssistant

  MusicAssistant is a SwiftUI iOS app for discovering songs, viewing track
  details, and saving favorites in a clean, mobile-first interface. It is built
  as a portfolio project to demonstrate practical iOS development fundamentals
  and product-level polish.
  Built solo using SwiftUI.

  Quick walkthrough

  1. Open the app and browse the song list on the Home screen.
  2. Tap a track to view detailed info.
  3. Save tracks to Favorites and revisit them anytime.

  ## Features

  - Browse a curated list of songs in a responsive SwiftUI interface.
  - View song-specific detail pages with key metadata.
  - Mark and unmark favorite songs.
  - Persist lightweight user preferences between launches.
  - Handle loading, empty, and basic error states with clear UI feedback.

  ## Skills

  - Architected the app using MVVM with clear separation of UI, state, and
    business logic.
  - Implemented async/await data fetching with Codable-based JSON decoding.
  - Built reusable SwiftUI components to keep views consistent and maintainable.
  - Managed app-wide and screen-level state using @StateObject, @ObservedObject,
    and @Published.
  - Used AppStorage for lightweight local persistence of user preferences.
  - Structured navigation flows with NavigationStack and typed destinations.
  - Added defensive error handling and fallback UI for unstable data/network
    scenarios.
  - Organized project files and modules for readability and portfolio-grade code
    review.

  ## Setup (Run in 2 Minutes)

  1. Clone the repository:

     git clone https://github.com/<your-username>/MusicAssistant.git
  2. Open the project in Xcode:

     open MusicAssistant.xcodeproj
  3. Select an iOS Simulator (or connected device) in Xcode.
  4. Press Run (Cmd + R).

  If the app uses local sample data, ensure the JSON/mock files are included in
  the app target (Target Membership in Xcode).
